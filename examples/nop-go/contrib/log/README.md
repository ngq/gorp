# contrib/log

该目录承接 gorp 的日志后端实现，当前真实完成态非常明确，应以“守边界、守口径”为主，而不是继续扩散候选后端数量。

## 当前状态口径

- `已完成`：`zap`

## 说明

- `zap` 是当前唯一明确落地的真实日志后端。
- `framework/provider/log` 保留的是 provider 门面与变量层，不属于这里的未完成 contrib。
- 当前不应把 `logrus`、`zerolog` 等候选方向写成“已支持日志后端矩阵”。

## P0 约束

- 文档只宣传当前真实完成的 `zap` 后端。
- 不因为候选讨论存在，就提前扩展对外支持口径。
