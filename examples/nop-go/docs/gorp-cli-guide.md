## 安装

### 从 GitHub 安装

```bash
# 安装最新版本
go install github.com/ngq/gorp/cmd/gorp@latest

# 安装指定版本
go install github.com/ngq/gorp/cmd/gorp@v0.1.0
```

### 从源码安装（开发环境）

```bash
cd D:/project/gorp
go install ./cmd/gorp
```

### 验证安装

```bash
gorp version
# 输出: version=x.x.x commit=xxx buildDate=xxx
```

## 核心命令

### gorp new - 创建新项目

**最常用命令**，创建项目骨架。

#### 基本用法

```bash
# 默认单服务项目（推荐）
gorp new

# Wire 单服务项目（依赖注入）
gorp new wire

# 多服务项目
gorp new multi

# 多服务 Wire 项目
gorp new multi-wire

# 从 GitHub Release 创建（需要联网）
gorp new from-release
```

#### 选项

```bash
gorp new [intent] [options]

选项:
  --template string      模板类型: golayout, golayout-wire, multi-flat, multi-flat-wire
  --backend string       ORM 后端: gorm, ent (默认: gorm)
  --with-db bool         是否包含数据库示例 (默认: true)
  --with-swagger bool    是否启用 Swagger (默认: true)
  --framework-version    框架版本，如 v0.1.0（设置后不生成 replace）
```

#### 模板矩阵

| 位置参数 | 等价模板 | 适用场景 |
|---------|---------|---------|
| 无（默认） | `golayout` | 单服务快速起步 |
| `wire` | `golayout-wire` | 单服务 + Wire 依赖注入 |
| `multi` | `multi-flat` | 多服务项目 |
| `multi-wire` | `multi-flat-wire` | 多服务 + Wire 依赖注入 |

#### 交互式创建

```bash
$ gorp new
请输入项目名称: my-project
请输入框架源码路径 (默认: D:/project/gorp): D:/project/gorp

创建项目成功，目录: D:/workspace/my-project

下一步:
  cd my-project
  go mod tidy
  go run ./cmd/app
```

#### 指定框架版本

```bash
# 使用 GitHub 发布版本（不生成 replace）
gorp new --framework-version v0.1.0

# 使用本地框架源码（生成 replace）
gorp new  # 默认行为
```

#### 生成的项目结构

```
my-project/
├── cmd/
│   └── app/
│       └── main.go           # 服务入口
├── internal/
│   ├── handler/              # HTTP 处理器
│   ├── data/                 # 数据模型
│   └── service/              # 业务逻辑
├── app/
│   ├── http/
│   │   └── middleware/       # 中间件
│   └── provider/             # 业务 Provider
│       └── runtime_provider/ # 运行时装配
├── config/
│   └── config.yaml           # 配置文件
├── docs/                     # 文档
├── go.mod                    # 依赖管理
└── Makefile                  # 构建脚本
```

### gorp proto - Proto/gRPC 代码生成

从 `.proto` 文件生成 Go 代码。

#### 命令

```bash
gorp proto gen
```

#### 工作流程

1. 扫描 `api/proto/**/*.proto` 目录
2. 调用 `protoc` 生成 `*.pb.go` 和 `*_grpc.pb.go`
3. 输出到 `api/proto/` 目录（source_relative）

#### 前置依赖

```bash
# 安装 protoc
# macOS: brew install protobuf
# Linux: apt install protobuf-compiler
# Windows: 从 https://github.com/protocolbuffers/protobuf/releases 下载

# 安装 Go 插件
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest
```

#### Proto 文件放置

```
api/proto/
├── user/
│   └── v1/
│       └── user.proto       # Proto 定义
│       ├── user.pb.go       # 生成的消息类型
│       └── user_grpc.pb.go  # 生成的 gRPC 代码
└── order/
    └── v1/
        └── order.proto
```

#### 示例 Proto

```protobuf
// api/proto/user/v1/user.proto
syntax = "proto3";

package user.v1;

option go_package = "github.com/your-project/api/proto/user/v1;v1";

service UserService {
    rpc GetUser(GetUserReq) returns (GetUserResp);
}

message GetUserReq {
    int64 id = 1;
}

message GetUserResp {
    int64 id = 1;
    string name = 2;
}
```

#### 执行生成

```bash
$ gorp proto gen
generated 2 proto files
```

### gorp model - 数据库模型生成

从数据库表结构生成 Go struct。

#### 子命令

```bash
gorp model gen    # 从表生成 struct
gorp model api    # 从 struct 生成 CRUD API
gorp model test   # 模型测试工具
```

#### gorp model gen

```bash
gorp model gen --table <表名> --out <输出目录>

选项:
  --table string  表名（必填）
  --out string    输出目录（默认: app/model/gen）
```

#### 示例

```bash
# 从 users 表生成 User struct
gorp model gen --table users

# 从 orders 表生成，指定输出目录
gorp model gen --table orders --out internal/data
```

#### 生成的代码

```go
// app/model/gen/user.go
package gen

// Code generated by gorp model gen; DO NOT EDIT.

type User struct {
    ID        int64  `db:"id" json:"id"`
    Name      string `db:"name" json:"name"`
    Email     string `db:"email" json:"email"`
    CreatedAt string `db:"created_at" json:"created_at"`
}
```

#### 类型映射

| SQLite 类型 | Go 类型 |
|------------|---------|
| INT, INTEGER | `int64` |
| CHAR, TEXT, CLOB | `string` |
| BLOB | `[]byte` |
| REAL, FLOAT, DOUBLE | `float64` |
| BOOL | `bool` |
| 其他 | `string` |

## 脚手架命令

### gorp provider - Provider 脚手架

创建新的业务 Provider 骨架。

#### 子命令

```bash
gorp provider list   # 列出已注册的 Provider
gorp provider new    # 创建新 Provider
```

#### gorp provider new

```bash
$ gorp provider new
请输入服务名称(服务凭证): payment
请输入服务所在目录名称(默认: 同服务名称): payment

创建服务成功, 文件夹地址: app/provider/payment
请不要忘记挂载新创建的服务
```

#### 生成的文件

```
app/provider/payment/
├── contract.go   # 接口定义和 Key
├── provider.go   # Provider 实现
└── service.go    # Service 实现
```

**contract.go**：
```go
package payment

const PaymentKey = "payment"

type Service interface {
    Foo() string
}
```

**provider.go**：
```go
package payment

import "github.com/ngq/gorp/framework/contract"

type PaymentProvider struct{}

func NewProvider() *PaymentProvider { return &PaymentProvider{} }

func (p *PaymentProvider) Name() string { return "payment" }
func (p *PaymentProvider) IsDefer() bool { return false }
func (p *PaymentProvider) Provides() []string { return []string{PaymentKey} }

func (p *PaymentProvider) Register(c contract.Container) error {
    c.Bind(PaymentKey, func(contract.Container) (any, error) {
        return NewService(), nil
    }, true)
    return nil
}

func (p *PaymentProvider) Boot(contract.Container) error { return nil }
```

**service.go**：
```go
package payment

type service struct{}

func NewService() Service {
    return &service{}
}

func (s *service) Foo() string { return "" }
```

#### 挂载 Provider

```go
// cmd/app/main.go 或 bootstrap 中
c.RegisterProviders(
    paymentprovider.NewProvider(),  // 添加新 Provider
)
```

### gorp middleware - 中间件脚手架

创建新的业务中间件骨架。

#### 子命令

```bash
gorp middleware list      # 列出已有中间件
gorp middleware new       # 创建新中间件
gorp middleware migrate   # 迁移 gin-contrib 中间件
```

#### gorp middleware new

```bash
$ gorp middleware new
请输入中间件名称: ratelimit

创建中间件成功, 文件夹地址: app/http/middleware/ratelimit
```

#### 生成的文件

```go
// app/http/middleware/ratelimit/middleware.go
package ratelimit

import "github.com/gin-gonic/gin"

// Middleware 返回该中间件的 gin.HandlerFunc。
func Middleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Next()
    }
}
```

#### 使用中间件

```go
// internal/handler/setup.go
import "your-project/app/http/middleware/ratelimit"

rt.Engine.Use(ratelimit.Middleware())
```

## 最佳实践

### 1. 新项目创建流程

```bash
# 1. 创建项目
gorp new my-project

# 2. 进入项目
cd my-project

# 3. 安装依赖
go mod tidy

# 4. 运行服务
go run ./cmd/app
```

### 2. 开发流程

```bash
# 添加新 Provider
gorp provider new
# 编辑生成的代码，补充业务逻辑

# 添加新中间件
gorp middleware new
# 编辑生成的代码，补充中间件逻辑

# 从数据库生成模型
gorp model gen --table users
# 基于生成的模型开发 CRUD

# 生成 Swagger 文档
gorp swagger gen
```

### 3. Proto/gRPC 开发流程

```bash
# 1. 编写 Proto 文件
vim api/proto/user/v1/user.proto

# 2. 生成 Go 代码
gorp proto gen

# 3. 实现 gRPC 服务
vim internal/grpc/user_service.go

# 4. 注册并运行
```

### 4. 代码生成工作流

```
数据库表 → gorp model gen → Go struct → 业务开发
                                ↓
                        gorp model api → CRUD API（可选）
```

## 总结

gorp CLI 的核心定位是 **工具链增强**，主要命令：

| 层级 | 命令 | 使用场景 |
|-----|------|---------|
| 核心 | `gorp new` | 创建项目 |
| 核心 | `gorp proto gen` | Proto/gRPC 生成 |
| 核心 | `gorp model gen` | 数据库模型生成 |
| 脚手架 | `gorp provider new` | Provider 脚手架 |
| 脚手架 | `gorp middleware new` | 中间件脚手架 |
| 文档 | `gorp swagger gen` | Swagger 文档 |

用户通过这些命令提升开发效率，服务启动通过项目自己的 `cmd/*/main.go` 完成。
